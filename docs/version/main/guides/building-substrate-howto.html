Quarkus - Building a Custom SubstrateVM

  • 前提条件
    • Install JVMCI
    • Install MX
  • Checkout and build SubstrateVM
  • カスタムGraalVMディストリビューションの構築

前提条件

Install JVMCI

何かをする前に、まずJVMCIビルドをインストールしなければなりません。

これらは ここでOTNからダウンロードすることができます

別のシステムにインストールする場合は、ChromeやFirefoxの開発者ツールを使用して、auth パラメーターを指定して直接 URL をキャプチャすることができます。

ダウンロードした後、インストールして JAVA_HOME をセットします

wget -O jvmci.tgz http://download.oracle.com/otn/utilities_drivers/oracle-labs/labsjdk-8u202-jvmci-0.56-darwin-amd64.tar.gz?AuthParam=[GENERATED AUTH TOKEN HERE]
tar -xzvf jvmci.tgz -C /opt
# On Mac do labsjdk1.8.0_202-jvmci-0.56/Contents/Home
export JAVA_HOME=/opt/labsjdk1.8.0_202-jvmci-0.46/
export PATH=$JAVA_HOME/bin:$PATH
注意

graalは最小要件を更新し続ける傾向があるので、最新版を入手しましょう。

Install MX

今、あなたはGraalの特別なビルドツール mx をインストールする必要があります、.

git clone https://github.com/graalvm/mx.git
export PATH=`pwd`/mx:$PATH

Checkout and build SubstrateVM

これでSubstrateのチェックアウトとビルドができるようになりました。

git clone https://github.com/oracle/graal.git
cd graal/substratevm
mx build

一度構築したら、その場でツールを素早く実行することができます。

echo "public class HelloWorld { public static void main(String[] args) { System.out.println(\"Hello World\"); } }" > HelloWorld.java
javac HelloWorld.java
mx native-image HelloWorld
./helloworld

カスタムGraalVMディストリビューションの構築

カスタムGraalVMディストリビューションを作成するためには、(quarkusで使用するために必要な)vmサブディレクトリーに切り替えて、特別なビルドオプションを使用して、ディストリビューションのモジュールを選択する必要があります。

substrate とそのネイティブツールだけのシンプルなサブセット VM の場合は、以下を実行します。

 cd ../vm
 mx --dy /substratevm,/tools build
 tar -czvf my-custom-graal.tgz -C latest_graalvm .

純正CEビルドの場合は以下のようになります。

mx --dy /substratevm,/tools,/sulong,/graal-nodejs build
tar -czvf my-custom-graal.tgz -C latest_graalvm .
